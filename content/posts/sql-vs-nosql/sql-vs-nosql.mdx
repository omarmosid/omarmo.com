## Data Structure
SQL databases are table based. They are inherently relational in nature.

No-SQL databases are document/graph based. They are non-relational in nature.

## Flexibility
SQL are not flexible when it comes to schema. They require you invest time and energy upfront before you start. 
Once you've decided the structure of your data, any change later down the line might be disruptive.

No-SQL databases are more flexible in nature. Since everything is a document, you can create them without having to first define stucture. 
There is no neccessity that every document should have its own unique structure. Any future changes to schema are easier to make.

## Scalability
SQL databases are vertically scalable. This means that as our dataset size increases(withing a single server) we can counter by simply adding more
RAM, CPU or SSD

No-SQL databases are horizontally scalable. This means that you handle an increased load/traffic by sharing your data or by increasing the number of servers
for your no-SQL database.

## Examples
Example of No-SQL database is MySQL, PostGres, etc.

Example of SQL database is MongoDB, Redix, Cassandra etc.
